# **­DHCP:**

**Dynamic Host Configuration Protocol (Protocolo de configuración dinámica de equipos)**

Protocolo que permite a los equipos de una red obtener direcciones IPs de forma automática asignada por el servidor

Funciona en la capa de Aplicación

**Servidor DHCP:** equipo que reparte las configuraciones

**Cliente DHCP:** equipo que solicita una configuración

**Puertos:**

**DHCP** utiliza los puertos **67** y **68** por defecto, para escuchar
y enviar.

El servidor DHCP almacena a quién le ha repartido una configuración y
hasta

Cuando se la concede. También tiene un rango de IP disponibles para
repartir

## **­­La configuración proporciona por servicio DHCP consiste en:**

**Intervalo de direcciones IP:** **Aquí debemos indicar el Intervalo/Conjunto de direcciones asignables
dentro de una red, desde el primer host asignable al ultimo**

**Exclusiones:** **Aquí podemos indicar las direcciones IP o intervalos de direcciones
que no queremos que sean asignadas a ningún equipo de la red**

**Concesión:** **Duración de la asignación y configuración IP a un equipo en concreto,
cuando el tiempo de la concesión acabe será asignada otra dirección IP
al equipo dejando libre la actual.**

**Enrutador o Puerta de enlace:** **En este apartado hay que indicar la IP del equipo que va a permitir a
nuestra red conectarse a otras**

**DNS y WINS:** **Son Protocolos de resolución de nombre de dominio, es decir hacen la
traducción IP-Nombre de dominio, permitiendo buscar un dominio por su
nombre y no su IP y viceversa**

**Reservas:** **Esta configuración nos permite asignarle una dirección IP especifica a
un equipo en concreto, para ello debemos especificar la dirección MAC
del cliente y la dirección IP que queremos asignarle.**
