## **WEB (ISS) Internet Information Service**
___
El servicio IIS o WEB tiene alojado documentos y páginas web, que responde a las peticiones de los clientes cuando estos solicitan acceder a ellos

Funciona en la capa de aplicación

En pocas palabras es un servicio que proporciona acceso a paginas web mediante el protocolo http

**Cliente:** Un cliente desde un navegador solicita.

**Servidor:** El servidor proporciona acceso al servicio solicitado por
el cliente.

### **Puertos:**
___
El protocolo **http** utiliza el puerto **80** por defecto

El protocolo **https** utiliza el puerto **443** por defecto

## **Conceptos:**
___
**ISS:**
Un servidor WEB o HTTP es un servidor que proporciona acceso a las aplicaciones, documentos, paginas web… alojadas en el mediante los protocolos como http/s, y smtp/s

**Hipertexto:**
documento en el que puede aparecer texto, elementos multimedia, programas y vínculos/links hacia otros documentos.

**HTML:**
Lenguaje para diseñar páginas web basado en marcas o etiquetas.

**HTTP:**
Protocolo de transferencia de hipertexto, como documentos y páginas web, sirve para la comunicación entre navegador y servidor. (Funciona en la capa de aplicación)

**Navegador:**
Programa intermediario entre el usuario cliente y el servidor que lanza peticiones HTTP y muestra las webs obtenidas, interpretando HTML

**WWW:**
(World wide web), conjunto de documentos hipertexto enlazados unos con otros

**WEB:**
Documento de hipertexto interpretado por un navegador

**URL (Uniform Resource Locator):**
Dirección que sirve para identificar un recurso dentro de una red (una foto, página web, fichero…)

## **Formato básico**
___
Protocolo://máquina/ruta/fichero

**Ejemplo:**

[<u>http://www.prueba.es/ficheros/ejemplo/foto1.jpg</u>](http://www.prueba.es/ficheros/ejemplo/foto1.jpg)

**Protocolo:** http

**Máquina:** www.prueba.es

**Ruta:** /ficheros/ejemplo

**Fichero:** foto1.jpg

En ‘protocolo’ y ‘máquina’, no se distingue entre mayúsculas y minúsculas. En el resto, depende del

servidor y del SO

## **Estructura del formato**
___
**Protocolo:** puede ser http, https, ftp…

**Máquina:** nombre completo del equipo o su IP. OBLIGATORIO

**Ruta:** localización del fichero dentro del servidor

**Fichero:** nombre completo y extensión del archivo

**Valores por defecto:**

**Protocolo** → http

**ruta** → raíz

**fichero** → index.html
